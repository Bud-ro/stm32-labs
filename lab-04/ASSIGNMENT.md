## Assignment

The original assignment was to develop on top of Module 3 (UART printf + LED blink). It has three parts.

### Part I (15%) - UART Driver with Register Manipulation

1. Replace the `MX_USART2_UART_Init()` function generated by the MX configurator with your own code that manipulates related registers manually. You can create a new function and name it `USR_USART2_UART_Init()`. (10%)
2. Replace the `HAL_UART_Transmit(&huart2, (uint8_t *)&ch, 1, HAL_MAX_DELAY);` line in the `__io_putchar()` implementation with your own implementation for transmitting single characters. Hint: Program logic to check if the UART FIFO has an available spot before sending a character to the Transmit Data Register (TDR). (5%)
3. Make a commit to your project at this point, ensuring it includes all the changes. (5%)

> Already implemented in the Zig BSP (`nucleo-g071rb/src/uart.zig`) - the driver was register-level from the start.

### Part II (40%) - Ring Buffer

Create a function to process the ring buffer, named `ring_add()`. You can use the pseudo code and the flow chart from the supported material section to assist.

1. Receive characters from the PC in ASCII code format through UART and store them into an array of fixed size. Declare a global array of fixed size, maybe 20 elements. Every time the user hits a key on the keyboard, Tera Term will send this keystroke to the MCU. Your code should be able to store this key into the array. (8%)
2. The ring buffer wraps around to the first spot when it reaches the last spot. Hint: When the array is full, overwrite the first element of the array. Currently, the element being overwritten will be lost, but we do not care. (8%)
3. Implement real-time character echo on Tera Term. This step allows the user to see what has been received on the MCU and should be based on steps 1 and 2. Hint: Simply send or print the received character back. (8%)
4. Remove a displayed character from the PC terminal as well as from the ring buffer when the Backspace key is pressed. Hint: If it is a backspace key, program a small sequence to erase the previous character stored in the ring buffer as well as visually remove it on Tera Term. Backspace is `\b`. (8%)
5. Upon hitting the Enter key, the terminal should display all the input characters, switch to a new line (`\n\r`), and display the entire content of the ring buffer, with each position being indexed. Hint: This is for debugging purposes so that you can clearly see what has been stored in the ring buffer. You can use a for loop to print each character in the ring buffer. (8%)
6. Make a commit to your project at this point, ensuring it includes all the changes. (5%)

### Part III (45%) - Command Parser

Create a function to implement the `parser()` function and another to reset the ring buffer index after parsing a command.

1. When a carriage return key is received and stored in the ring buffer, the `parser()` function is called to parse the received string. If there are multiple strings or commands received in the ring buffer, they could be separated by `\r`. The grader will look for code that implements this logic. (10%)
2. Define three commands: `STOP\r`, `START\r`, and `CLEAR\r` as either strings or arrays. When they are typed in Tera Term, the parser returns a unique number. It could be 0 for STOP, 1 for START, etc. For example, when the user types `START\r` in Tera Term, the MCU should be able to recognize it and reply "START command is received in Tera Term". (15%)
3. The parser returns an error message when an undefined message is received, or the ring buffer overflows with more characters being received. For example, if "CLEAN" is typed, it may reply with "undefined command". (10%)
4. The parser can continuously parse new commands. (10%)
5. Make a commit to your project at this point, ensuring it includes all the changes. (5%)
